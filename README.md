# Collab Board

Коллаборативная интерактивная доска в реальном времени — командный проект по дисциплине «Разработка кода информационных систем».

Несколько пользователей могут подключиться к одной доске по её ID и совместно создавать, редактировать, перемещать и удалять графические объекты — все изменения синхронизируются через WebSocket.

---

## Реализованные фичи

- [x] Создание новой доски (REST: `POST /board`)
- [x] Подключение нескольких пользователей по ID доски
- [x] Реальновременная синхронизация через WebSocket
- [x] 6 типов объектов: **текст · изображение · линия · прямоугольник · треугольник · круг**
- [x] Редактирование свойств (координаты, размер, цвет, толщина линии, заливка, текст, шрифт)
- [x] Перемещение объектов мышкой
- [x] Удаление объектов
- [x] Индикатор подключения и счётчик пользователей онлайн
- [x] Горячие клавиши (V/R/C/T/L/X/I/Del)
- [x] Автопереподключение WebSocket при разрыве

---

## Стек технологий

**Backend**
- Node.js
- Express 4
- ws (WebSocket)
- Хранение состояния in-memory (Map)

**Frontend**
- Vanilla JavaScript (ES2020+)
- SVG — интерактивный холст
- CSS (тёмная тема, GitHub-inspired)
- Без build-шага

## Запуск

**Требования:** Node.js 18+

```bash
cd server
npm install
npm start
```

Открыть в браузере: **http://localhost:3000**

Для проверки совместной работы — открой доску в двух разных окнах/устройствах и подключайся по одному ID.

## Архитектура

```
┌──────────────────┐                    ┌──────────────────┐
│   Клиент A       │ ◄── WebSocket ──►  │                  │
│   (браузер)      │                    │                  │
├──────────────────┤                    │   Node.js        │
│   Клиент B       │ ◄── WebSocket ──►  │   Express + ws   │
│   (браузер)      │                    │                  │
├──────────────────┤                    │   ┌────────────┐ │
│   Клиент C       │ ◄──── REST ─────►  │   │ Map<id,    │ │
│   (браузер)      │   POST /board      │   │  Board>    │ │
└──────────────────┘                    │   └────────────┘ │
                                         └──────────────────┘
```

**Поток данных при совместной работе:**
1. Клиент создаёт объект → шлёт `create-object` по WS
2. Сервер сохраняет объект в `Map` текущей доски
3. Сервер рассылает `create-object` всем остальным клиентам этой доски
4. Клиенты рендерят объект в SVG

**Поддерживаемые WS-события:**

| Событие            | Направление     | Описание                         |
|--------------------|-----------------|----------------------------------|
| `join-board`       | client → server | Подключение к доске              |
| `board-state`      | server → client | Текущее состояние + userId       |
| `user-joined/left` | server → client | Изменение числа пользователей    |
| `create-object`    | ↔               | Создание объекта                 |
| `update-object`    | ↔               | Изменение свойств                |
| `move-object`      | ↔               | Перемещение                      |
| `delete-object`    | ↔               | Удаление                         |

## Примечания для расширения

- Состояние в памяти — после перезапуска сервера доски сбрасываются. Для прод-варианта подключить Redis или SQLite.
- Картинки сейчас хранятся как data-URL в памяти сервера. Для больших файлов — выгрузка на диск/S3.
- Нет авторизации — любой со знанием ID может войти. Для прод — JWT + права доступа.
